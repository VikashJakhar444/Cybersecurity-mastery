# SS Command (Socket Statistics)

--------------------------------------------------
1. Introduction
--------------------------------------------------

The `ss` (Socket Statistics) command is a Linux networking utility used to
display information about sockets.

A socket is simply an endpoint of communication between two programs.

Whenever your system:

✔ Opens a website
✔ Resolves DNS
✔ Uses SSH
✔ Downloads a file
✔ Connects to a VPN
✔ Uses DHCP

A socket is created.

The ss command allows us to inspect these sockets.

Unlike netstat, ss reads information directly from the Linux kernel, making it
much faster and more efficient.

In this chapter, we will only study the commands that we actually practiced.

--------------------------------------------------
2. Why We Use It
--------------------------------------------------

We use ss to:

✔ View active network connections.

✔ Check whether a service is listening.

✔ View TCP sockets.

✔ View UDP sockets.

✔ View IPv4 sockets.

✔ View IPv6 sockets.

✔ Display process names using sockets.

✔ Display all sockets.

✔ Troubleshoot networking problems.

✔ Verify whether an application is connected to another system.

--------------------------------------------------

--------------------------------------------------

Basic Syntax

ss [OPTION]

Examples

ss

ss -t

ss -u

ss -l

ss -a

8. Interview Questions
--------------------------------------------------

Q1

What does ss stand for?

Answer

Socket Statistics.

------------------------------------------

Q2

Why is ss preferred over netstat?

Answer

Because it reads socket information directly
from the Linux kernel and is much faster.

------------------------------------------

Q3

Difference between TCP and UDP output?

Answer

TCP normally shows states like

LISTEN

ESTAB

UDP usually appears as

UNCONN

because it is connectionless.

------------------------------------------

Q4

What does ESTAB mean?

Answer

Established.

Data can currently be exchanged.

------------------------------------------

Q5

What does LISTEN mean?

Answer

The application is waiting
for incoming connections.

------------------------------------------

Q6

What does UNCONN mean?

Answer

No active connection.

Common for UDP,
Netlink and ICMP sockets.

------------------------------------------

Q7

Why did ss -t show nothing?

Answer

Because no active TCP connection existed
at that moment.

------------------------------------------

Q8

Why did ss -u show output?

Answer

Background services like

DNS

DHCP

Cloudflare WARP

were using UDP.

------------------------------------------

Q9

Why did ss -6 show only icmp6?

Answer

No active IPv6 TCP/UDP sockets existed.

Only the ICMPv6 socket was active.

------------------------------------------

Q10

What does -n do?

Answer

Displays numeric port numbers instead of service names.

------------------------------------------

Q11

What does -p do?

Answer

Shows the process using each socket.

------------------------------------------

Q12

Why did ss '( sport = :22 )'
return nothing?

Answer

No socket matched source Port 22.

------------------------------------------

Q13

Why did ss '( dport = :80 )'
return nothing?

Answer

No socket was communicating
with destination Port 80.

------------------------------------------

Q14

Why did ss '( dport = :80)'
produce an error?

Answer

Incorrect filter syntax.

------------------------------------------

Q15

Why did ss src <ip>
produce a shell error?

Answer

Because "<ip>" is only a placeholder.
A real IP address must be supplied.
