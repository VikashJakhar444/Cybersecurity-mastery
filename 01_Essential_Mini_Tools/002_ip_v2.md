# 1. Introduction

In this chapter, we move from **viewing** network information to **modifying** it using the `ip` command.

Instead of changing the real network interface, we perform all experiments on a **Dummy Interface**, allowing us to safely practice creating interfaces, assigning IP addresses, enabling/disabling interfaces, and deleting configurations.

---

# 2. Why We Use It

The `ip` command allows administrators to:

- Create virtual interfaces
- Enable or disable interfaces
- Assign IPv4 addresses
- Assign IPv6 addresses
- Configure multiple IPs on one interface
- Remove IP addresses
- Flush all addresses
- Delete interfaces
- Verify routing
- Monitor live networking events

---

# 3. Syntax

Create Interface

```bash
sudo ip link add <interface> type dummy
```

Bring Interface UP

```bash
sudo ip link set <interface> up
```

Bring Interface DOWN

```bash
sudo ip link set <interface> down
```

Assign IPv4

```bash
sudo ip addr add IP/CIDR dev <interface>
```

Assign IPv6

```bash
sudo ip -6 addr add IPv6/CIDR dev <interface>
```

Delete Address

```bash
sudo ip addr del IP/CIDR dev <interface>
```

Flush Addresses

```bash
sudo ip addr flush dev <interface>
```

Delete Interface

```bash
sudo ip link delete <interface>
```

# 5. Basic Examples

## Example 1 — Create Dummy Interface

Command

```bash
sudo ip link add dummy0 type dummy
```

Meaning

Creates a virtual network interface named **dummy0**.

A Dummy Interface is not connected to any physical hardware and is commonly used for testing, routing, and lab environments.

---

## Example 2 — View Interface

Command

```bash
ip link show dummy0
```

Meaning

Displays information about the newly created interface.

If the state is `DOWN`, the interface exists but is inactive.

---

## Example 3 — Brief Interface View

Command

```bash
ip -br link
```

Meaning

Displays a short summary of all interfaces and their current state.

Useful for quickly checking whether an interface is UP or DOWN.

---

## Example 4 — Bring Interface UP

Command

```bash
sudo ip link set dummy0 up
```

Meaning

Activates the interface.

Dummy interfaces usually show the state `UNKNOWN` because they have no physical cable or carrier.

---

## Example 5 — Assign IPv4 Address

Command

```bash
sudo ip addr add 192.168.100.10/24 dev dummy0
```

Meaning

Assigns an IPv4 address to the interface.

The interface becomes part of the `192.168.100.0/24` network.

---

## Example 6 — Assign a Second IPv4 Address

Command

```bash
sudo ip addr add 192.168.100.20/24 dev dummy0
```

Meaning

Adds another IPv4 address to the same interface.

Linux supports multiple IP addresses on a single interface.

The first address is the **Primary** address, while additional ones are marked as **Secondary**.

---

## Example 7 — Assign IPv6 Address

Command

```bash
sudo ip -6 addr add fd00::10/64 dev dummy0
```

Meaning

Assigns an IPv6 address to the interface.

The interface now supports both IPv4 and IPv6 (Dual Stack).

---

## Example 8 — Bring Interface DOWN

Command

```bash
sudo ip link set dummy0 down
```

Meaning

Disables the interface.

The interface configuration remains stored, but traffic is no longer transmitted or received.

---

## Example 9 — Bring Interface UP Again

Command

```bash
sudo ip link set dummy0 up
```

Meaning

Re-enables the interface.

Previously assigned addresses remain intact.

---

## Example 10 — Delete an IPv4 Address

Command

```bash
sudo ip addr del 192.168.100.20/24 dev dummy0
```

Meaning

Removes only the specified IP address.

Other configured addresses remain unchanged.

---

## Example 11 — Flush All Addresses

Command

```bash
sudo ip addr flush dev dummy0
```

Meaning

Removes every IPv4 and IPv6 address from the interface.

The interface itself is **not deleted**.

---

## Example 12 — Delete Interface

Command

```bash
sudo ip link delete dummy0
```

Meaning

Completely removes the interface from the system.

---

## Example 13 — Interface Statistics

Command

```bash
ip -s addr show eth0
```

Meaning

Displays address information along with traffic statistics such as:

- RX packets
- TX packets
- Errors
- Dropped packets

Useful for troubleshooting network problems.

---

## Example 14 — Detailed Interface Information

Command

```bash
ip -d addr show eth0
```

Meaning

Shows advanced interface details like MTU, queues, promiscuity, and driver information.

Useful for advanced debugging.

---

## Example 15 — Route Lookup

Command

```bash
ip route get 8.8.8.8
```

Meaning

Shows how Linux will route packets to a destination.

Displays:

- Destination
- Gateway
- Outgoing interface
- Source IP

---

## Example 16 — View Neighbor Table

Command

```bash
ip neigh
```

Meaning

Displays the ARP/NDP neighbor cache.

Common neighbor states include:

- `REACHABLE`
- `STALE`
- `DELAY`
- `PROBE`

---

## Example 17 — Live Monitoring

Command

```bash
ip monitor
```

Meaning

Displays live network events such as interface, address, route, and neighbor changes in real time.

# 8. Interview Questions

**Q. Why do we use a Dummy Interface?**

A. To safely test networking without affecting physical interfaces.

---

**Q. What is the difference between DOWN and DELETE?**

A. `DOWN` disables the interface temporarily, while `DELETE` removes it completely.

---

**Q. Can one interface have multiple IP addresses?**

A. Yes. Linux supports multiple (primary and secondary) IP addresses on a single interface.

---

**Q. What does `ip addr flush` do?**

A. It removes all IP addresses from an interface but keeps the interface itself.

---

**Q. What is `ip monitor` used for?**

A. To watch live network events in real time.

---

**Q. What is the difference between `ip addr del` and `ip addr flush`?**

A. `del` removes one specific address, while `flush` removes all addresses from the interface.

---

**Q. Why does a Dummy Interface usually show `UNKNOWN`?**

A. Because it has no physical carrier or cable attached.
