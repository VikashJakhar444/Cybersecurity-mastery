> Category: Essential Linux Networking Tools  
> Difficulty: ⭐ Beginner

---

# 1. Introduction

`ip` is the modern Linux networking command-line utility from the **iproute2** package. It replaces older tools like `ifconfig`, `route`, and `arp`.

It is used to inspect network interfaces, IP addresses, routing tables, neighbors, and interface statistics.

> **Note:** This version covers only read-only commands. No command modifies the system.

---

# 2. Why We Use It

The `ip` command is commonly used to:

- View network interfaces
- Check IPv4 & IPv6 addresses
- Display routing tables
- Find the default gateway
- View neighbor (ARP/NDP) entries
- Check interface status
- Monitor traffic statistics
- Troubleshoot networking issues
- Generate JSON output for automation

---

# 3. Syntax

Basic Syntax

```bash
ip [OPTIONS] OBJECT COMMAND
```

Examples

```bash
ip addr
ip link
ip route
ip neigh
```

---

# 4. Common Options

| Option | Description |
|---------|-------------|
| `-4` | Show IPv4 only |
| `-6` | Show IPv6 only |
| `-br` | Brief output |
| `-s` | Show statistics |
| `-d` | Detailed output |
| `-j` | JSON output |
| `-p` | Pretty JSON |
| `help` | Show help |

# 5. Basic Examples

## Example 1 — Check Version

Command

```bash
ip -V
```

Meaning

Displays the installed version of the `ip` utility.

Command

- tunnel

---

## Example 3 — Locate Binary

Command

```bash
which ip
```

Meaning

Shows where the `ip` command is installed or whether it is an alias.

---

## Example 4 — Show Network Interfaces

Command

```bash
ip link show
```

Meaning

Displays all network interfaces and their status.

Common flags include:

- `UP` → Interface enabled
- `LOWER_UP` → Physical link available
- `LOOPBACK` → Internal interface
- `BROADCAST` → Supports broadcast
- `MULTICAST` → Supports multicast
- `NOARP` → ARP disabled
- `POINTOPOINT` → Point-to-point interface

---

## Example 5 — Show IP Addresses

Command

```bash
ip addr show
```

Meaning

Displays all IPv4 and IPv6 addresses assigned to every interface.

---

## Example 6 — Show IPv4 Only

Command

```bash
ip -4 addr
```

Meaning

Displays only IPv4 addresses.

---

## Example 7 — Show IPv6 Only

Command

```bash
ip -6 addr
```

Meaning

Displays only IPv6 addresses.

---

## Example 8 — Brief Address View

Command

```bash
ip -br addr
```

Meaning

Shows a clean summary of interfaces and assigned IP addresses.

Widely used for quick troubleshooting.

---

## Example 9 — Brief Interface Status

Command

```bash
ip -br link
```

Meaning

Displays interface names and their current state (UP/DOWN).

---

## Example 10 — View Routing Table

Command

```bash
ip route
```

Meaning

Displays the system routing table.

Useful for identifying:

- Default route
- Default gateway
- Outgoing interface

---

## Example 11 — Route Lookup

Command

```bash
ip route get 8.8.8.8
```

Meaning

Shows how the kernel will route packets to a destination.

Displays:

- Destination
- Gateway
- Outgoing interface
- Source IP

---

## Example 12 — View Neighbor Table

Command

```bash
ip neigh
```

Meaning

Displays the ARP/NDP neighbor table.

Common states:

- `REACHABLE`
- `STALE`
- `PROBE`

---

## Example 13 — Interface Statistics

Command

```bash
ip -s link
```

Meaning

Displays interface traffic statistics such as:

- RX packets
- TX packets
- Bytes
- Errors
- Dropped packets

---

## Example 14 — Detailed Interface Information

Command

```bash
ip -d link show eth0
```

Meaning

Shows advanced interface details including MTU, queues, and driver information.

Useful for advanced troubleshooting.

---

## Example 15 — JSON Output

Command

```bash
ip -j addr
```

Meaning

Displays address information in JSON format.

Useful for scripting and automation.

Pretty JSON

```bash
ip -j -p addr
```

---

## Example 16 — Address Statistics

Command

```bash
ip -s addr
```

Meaning

Displays IP address information together with traffic statistics.

---

## Example 17 — Detailed Routes

Command

```bash
ip -d route
```

Meaning

Shows detailed routing information including protocol, metric, and route type.

---

## Example 18 — IPv6 Routes

Command

```bash
ip -6 route
```

Meaning

Displays only IPv6 routing entries.

---

## Example 19 — Neighbor Statistics

Command

```bash
ip -s neigh
```

Meaning

Displays neighbor entries along with state and usage statistics.

---

## Example 20 — Show Specific Interface

Command

```bash
ip addr show eth0
```

Meaning

Displays configuration for a single network interface.

---

## Example 21 — Monitor Network Changes

Command

```bash
ip monitor
```

Meaning

Displays live networking events such as address, route, or neighbor changes.

---

## Example 22 — Help for Address Object

Command

```bash
ip addr help
```

Meaning

Displays all available subcommands for the `addr` object.
