# IP Command (Version 2 - Interface & Address Modification)

## 1. Introduction

In this chapter, we will learn the practical modification part of the `ip` command.

Here, we will create live interfaces, bring them UP/DOWN, assign IP addresses, configure multiple IPs, assign IPv6 addresses, remove addresses, and finally delete the interface.

These operations are commonly performed on production Linux servers, so we practiced everything on a **Dummy Interface** to avoid affecting the real network.

# 3. Syntax

### Create Interface

```bash
sudo ip link add <interface> type dummy
```

### Bring Interface UP

```bash
sudo ip link set <interface> up
```

### Bring Interface DOWN

```bash
sudo ip link set <interface> down
```

### Assign IPv4

```bash
sudo ip addr add IP/CIDR dev <interface>
```

### Assign IPv6

```bash
sudo ip -6 addr add IPv6/CIDR dev <interface>
```

### Delete Address

```bash
sudo ip addr del IP/CIDR dev <interface>
```

### Remove All Addresses

```bash
sudo ip addr flush dev <interface>
```

### Delete Interface

```bash
sudo ip link delete <interface>
```

# 5. Basic Examples

## Example 1

### Create Dummy Interface

**Command**

```bash
sudo ip link add dummy0 type dummy
```

**Output**

(No output)

**Verification**

```bash
ip link show dummy0
```

```
5: dummy0: <BROADCAST,NOARP> mtu 1500 qdisc noop state DOWN mode DEFAULT group default qlen 1000
link/ether 32:87:5a:80:e0:9c brd ff:ff:ff:ff:ff:ff
```

**Explanation**

A Dummy Interface is a virtual network interface.

It is not connected to any physical cable or hardware.

It is mainly used for testing, labs, services binding, routing experiments, and production applications.

---

## Example 2

### Interface State

**Command**

```bash
ip link show dummy0
```

**Output**

```
state DOWN
```

**Explanation**

The interface has been created successfully.

However, it is currently inactive.

DOWN means Linux knows the interface exists, but it will not transmit or receive packets.

---

## Example 3

### Common Mistake

**Command**

```bash
ip -b link
```

**Output**

```
Cannot open file "link" for reading
```

**Reason**

`-b` means **Batch Mode**.

Linux treated `link` as a filename instead of a command.

**Correct Command**

```bash
ip -br link
```

**Learning**

Always remember:

```
-b  = Batch Mode

-br = Brief Output
```

---

## Example 4

### Bring Interface UP

**Command**

```bash
sudo ip link set dummy0 up
```

**Output**

(No output)

**Verification**

```bash
ip -br link
```

```
dummy0 UNKNOWN
```

**Explanation**

Linux usually prints nothing when a command executes successfully.

`UNKNOWN` is completely normal for Dummy Interfaces because they have no physical carrier.

---

## Example 5

### Assign First IPv4 Address

**Command**

```bash
sudo ip addr add 192.168.100.10/24 dev dummy0
```

**Output**

(No output)

**Verification**

```bash
ip addr show dummy0
```

```
inet 192.168.100.10/24 scope global dummy0
```

**Explanation**

The interface now owns a valid IPv4 address.

It has become part of the `192.168.100.0/24` subnet.

---

## Example 6

### Assign Second IPv4 Address

**Command**

```bash
sudo ip addr add 192.168.100.20/24 dev dummy0
```

**Verification**

```bash
ip addr show dummy0
```

```
inet 192.168.100.10/24 scope global dummy0
inet 192.168.100.20/24 scope global secondary dummy0
```

**Explanation**

Linux allows multiple IP addresses on a single interface.

The first IP is the **Primary Address**.

Additional IPs are marked as **Secondary**.

This is very common on web servers and load balancers.

---

## Example 7

### Wrong IPv6 Command

**Command**

```bash
sudo ip -6 add fd00::10/64 dev dummy0
```

**Output**

```
Command "fd00::10/64" is unknown
```

**Reason**

The `addr` keyword was missing.

**Correct Command**

```bash
sudo ip -6 addr add fd00::10/64 dev dummy0
```

**Learning**

The order of keywords in the `ip` command is important.

---

## Example 8

### Assign IPv6 Address

**Verification**

```bash
ip -6 addr show dummy0
```

```
inet6 fd00::10/64
```

**Explanation**

The interface now supports both IPv4 and IPv6.

This is known as a **Dual Stack Configuration**.

---

## Example 9

### Bring Interface DOWN

**Command**

```bash
sudo ip link set dummy0 down
```

**Output**

```
state DOWN
```

**Explanation**

The interface becomes inactive.

The configuration is still preserved.

Only the operational state changes.

---

## Example 10

### Bring Interface UP Again

**Command**

```bash
sudo ip link set dummy0 up
```

**Verification**

```
state UNKNOWN
```

**Explanation**

The previous configuration is restored automatically.

Only the interface state was changed.

---

## Example 11

### Delete Secondary IPv4 Address

**Command**

```bash
sudo ip addr del 192.168.100.20/24 dev dummy0
```

**Explanation**

Only the secondary IP is removed.

The primary address remains configured.

---

## Example 12

### Flush Interface Addresses

**Command**

```bash
sudo ip addr flush dev dummy0
```

**Explanation**

This removes all IPv4 and IPv6 addresses from the interface in one command.

The interface itself is not deleted.

---

## Example 13

### Delete Interface

**Command**

```bash
sudo ip link delete dummy0
```

**Explanation**

The Dummy Interface is completely removed from the system.

It will no longer appear in `ip link`.

---

## Example 14

### Interface Statistics

**Command**

```bash
ip -s addr show eth0
```

**Explanation**

Useful fields include:

- RX (Received Traffic)
- TX (Transmitted Traffic)
- Errors
- Dropped Packets
- Carrier Errors

These counters are commonly used during network troubleshooting.

---

## Example 15

### Detailed Interface Information

**Command**

```bash
ip -d addr show eth0
```

**Explanation**

Displays advanced interface details such as:

- Promiscuity
- Queue Information
- MTU Limits
- Driver Capabilities
- Hardware Details

Useful for advanced debugging.

---

## Example 16

### Route Lookup

**Command**

```bash
ip route get 8.8.8.8
```

**Output**

```
8.8.8.8 dev CloudflareWARP table 65743 src 172.16.0.2
```

**Explanation**

Initially, traffic was routed through **eth0**.

After enabling **Cloudflare WARP**, Linux automatically changed the routing path.

This is a real-world example of dynamic route selection.

---

## Example 17

### Neighbour Cache

**Command**

```bash
ip neigh
```

**Explanation**

Displays IP-to-MAC address mappings stored in the ARP/Neighbour cache.

Common neighbour states include:

- REACHABLE
- STALE
- DELAY
- PROBE

Very useful during ARP troubleshooting.

---

## Example 18

### Live Monitoring

**Command**

```bash
ip monitor
```

**Explanation**

Shows live network events as they happen.

During the lab, the neighbour state changed like this:

```
STALE
↓
PROBE
↓
REACHABLE
```

This demonstrates the normal neighbour state transition inside the Linux kernel.
