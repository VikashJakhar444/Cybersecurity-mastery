# 1. Introduction

`dig` (Domain Information Groper) is a command-line DNS lookup utility used to query DNS servers and retrieve DNS records of a domain.

Unlike `ping`, `dig` **does not test connectivity**. It only asks a DNS server for information about a domain.

It can retrieve records like:

- A (IPv4 Address)
- AAAA (IPv6 Address)
- MX (Mail Server)
- NS (Name Server)
- TXT (Text Records)
- CNAME (Alias)
- PTR (Reverse Lookup)

Simple Flow

Your PC

↓

DNS Query

↓

DNS Server

↓

DNS Response

↓

Your PC

---

# 2. Why We Use It?

`dig` is commonly used by Network Engineers, System Administrators and Cybersecurity Professionals.

Uses

- Check whether DNS is working.
- Find the IP address of a domain.
- Check Mail Servers (MX).
- Find Name Servers (NS).
- Verify TXT records.
- Perform Reverse DNS Lookup.
- Query a specific DNS server.
- Troubleshoot DNS problems.

---

# 3. Syntax

Basic Syntax

```bash
dig <domain>
```

Query a specific record

```bash
dig <domain> <Record_Type>
```

Query a specific DNS Server

```bash
dig @<DNS_Server> <domain>
```

Reverse Lookup

```bash
dig -x <IP_Address>
```

## Example 4 — Default DNS Lookup

Command

```bash
dig google.com
```

Important Output

```text
QUESTION SECTION:
google.com. IN A
```

Meaning

Your computer asked:

> "Give me the IPv4 address of google.com."

---

```text
ANSWER SECTION
```

Contains the answer returned by the DNS server.

Example

```text
google.com. 223 IN A 192.178.177.139
```

Explanation

| Field | Meaning |
|--------|----------|
| google.com | Domain Name |
| 223 | TTL (cache time in seconds) |
| IN | Internet Class |
| A | IPv4 Record |
| 192.178.177.139 | IPv4 Address |

---

```text
status: NOERROR
```

DNS query completed successfully.

---

```text
Query time: 36 ms
```

Time taken by the DNS server to respond.

Lower is generally better.

---

```text
SERVER: 127.0.2.2#53
```

DNS Server used.

127.0.2.2 → Resolver

53 → DNS Port

# 10. Summary

- `dig` is a DNS lookup utility.
- It queries DNS servers for domain information.
- `A` returns IPv4 addresses.
- `AAAA` returns IPv6 addresses.
- `MX` returns mail servers.
- `NS` returns authoritative name servers.
- `TXT` returns text and verification records.
- `CNAME` checks alias records.
- `PTR` performs reverse DNS lookup.
- `@DNS_Server` queries a specific DNS server.
- `+short` shows only the answer.
- `+trace` displays the complete DNS resolution path.
