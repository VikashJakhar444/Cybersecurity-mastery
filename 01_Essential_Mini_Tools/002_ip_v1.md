# IP Command

---

# 1. Introduction

`ip` Linux ka modern networking command-line utility hai jo **iproute2 package** ka part hai. Ye purane `ifconfig`, `route`, `arp` aur `netstat` jaise tools ka replacement hai.

Cybersecurity, System Administration aur Networking me `ip` sabse important commands me se ek hai kyunki isi se hum:

- Network Interfaces dekhte hain.
- IP Addresses check karte hain.
- Routing Table analyze karte hain.
- Gateway identify karte hain.
- Neighbor (ARP/NDP) entries dekhte hain.
- Network troubleshooting karte hain.
- Interface statistics monitor karte hain.

> **Version 1 Note:** Is handbook me sirf **Read-Only** commands cover kiye gaye hain. Koi bhi command system configuration change nahi karti.

---

# 2. Why We Use It

`ip` command ka use mainly networking ko inspect aur troubleshoot karne ke liye hota hai.

## Common Uses

- Check network interfaces
- Check IPv4 & IPv6 addresses
- View routing table
- Find default gateway
- View MAC Address
- Check interface status
- View neighbor (ARP) table
- View packet statistics
- Monitor live network changes
- JSON output for automation

---

# 3. Syntax

```bash
ip [OPTIONS] OBJECT COMMAND
```

Examples

```bash
ip addr

ip link

ip route

ip neigh
```

General Syntax

```bash
ip OBJECT COMMAND
```

Example

```bash
ip addr show
```

Where

```
ip      → Main command

addr    → Object

show    → Action
```

---

# 4. Common Options

| Option | Meaning |
|---------|----------|
| -4 | Show only IPv4 |
| -6 | Show only IPv6 |
| -br | Brief Output |
| -s | Statistics |
| -d | Detailed Output |
| -j | JSON Output |
| -p | Pretty JSON |
| help | Show Help |

# 5. Basic Examples

---

## Check Version

```bash
ip -V
```

Output

```text
ip utility, iproute2-v7.0.0, libbpf 1.7.0
```

Explanation

- Shows installed ip utility version.
- Useful while debugging compatibility issues.

---

## Help

```bash
ip -help
```

Output

(Your system displayed all available Objects and Options.)

Important Objects

```
address

link

route

neighbor

monitor

rule

tunnel
```

Meaning

Every networking task belongs to an object.

Example

```
Address → IP Address

Link → Interface

Route → Routing Table

Neighbor → ARP Table
```

---

## Check Binary

Command

```bash
which ip
```

Output

```text
ip: aliased to ip --color=auto
```

Explanation

Shows which executable/shell alias is being used.

# Lab 1 — List Interfaces

Command

```bash
ip link show
```

Output

```text
1: lo: <LOOPBACK,UP,LOWER_UP>
2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP>
3: CloudflareWARP: <POINTOPOINT,MULTICAST,NOARP,UP,LOWER_UP>
```

Explanation

### lo

Internal communication.

Primary Ethernet Interface.

---

### CloudflareWARP

VPN Interface.

# Lab 18 — Live Monitoring

Command

```bash
ip monitor
```

Observation

While monitoring, your system showed transitions like:

```text
PROBE
REACHABLE
```

Meaning:

- Kernel started verifying a neighbor (`PROBE`)
- Neighbor responded successfully (`REACHABLE`)

This command is useful for watching live networking events.
