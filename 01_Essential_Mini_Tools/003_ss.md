# 1. Introduction

`ss` (Socket Statistics) is a Linux networking utility used to display information about network sockets.

A socket is an endpoint of communication between two processes or between a process and a remote host.

Unlike `netstat`, the `ss` command reads socket information directly from the Linux kernel, making it much faster and more efficient.

It is commonly used to inspect active connections, listening ports, socket states, and the processes using them.

---

# 2. Why We Use It

The `ss` command is used to:

- View active TCP and UDP connections.
- Check listening ports.
- Display IPv4 and IPv6 sockets.
- Show process ownership of sockets.
- View all sockets on the system.
- Troubleshoot networking issues.
- Verify whether a service is listening.

---

# 3. Syntax

Basic Syntax

```bash
ss [OPTION]
```

Examples

```bash
ss
ss -t
ss -u
ss -l
ss -a
ss -n
ss -p
ss -4
ss -6
ss '( sport = :22 )'
ss '( dport = :80 )'
```

---

# 4. Common Options

| Option | Description |
|---------|-------------|
| `-t` | Show TCP sockets only |
| `-u` | Show UDP sockets only |
| `-l` | Show listening sockets |
| `-a` | Show all sockets |
| `-n` | Show numeric port numbers |
| `-p` | Show process using the socket |
| `-4` | Show IPv4 sockets only |
| `-6` | Show IPv6 sockets only |

# 5. Basic Examples

## Example 1 — Locate the Command

Command

```bash
which ss
```

Example Output

```text
/usr/bin/ss
```

Meaning

- Shows where the `ss` executable is installed.
- If a path is returned, the command is installed correctly.

---

## Example 2 — Check Version

Command

```bash
ss -V
```

Example Output

```text
ss utility, iproute2-v7.0.0
```

Meaning

Displays the installed version of the `ss` utility.

---

## Example 3 — View Help

Command

```bash
ss --help
```

Meaning

Displays all supported options available for the installed version.

---

## Example 4 — Show TCP Sockets

Command

```bash
ss -t
```

Meaning

Displays only TCP sockets.

If no TCP connection exists, only the table header is shown.

---

## Example 5 — Show UDP Sockets

Command

```bash
ss -u
```

Meaning

Displays active UDP sockets.

Common background services include:

- DNS
- DHCP
- VPN (Cloudflare WARP)

---

## Example 6 — Show Listening Sockets

Command

```bash
ss -l
```

Meaning

Displays sockets waiting for incoming connections.

Common socket types:

- TCP
- UDP
- UNIX
- Netlink
- ICMP

---

## Example 7 — Listening TCP Sockets

Command

```bash
ss -lt
```

Meaning

Shows only TCP services currently listening.

Useful for checking whether a server has started successfully.

---

## Example 8 — Listening UDP Sockets

Command

```bash
ss -lu
```

Meaning

Shows listening UDP sockets.

Since UDP is connectionless, sockets usually appear as `UNCONN`.

---

## Example 9 — Numeric TCP Output

Command

```bash
ss -tn
```

Meaning

Shows TCP sockets using numeric port numbers instead of service names.

---

## Example 10 — Numeric UDP Output

Command

```bash
ss -un
```

Meaning

Displays UDP sockets with numeric ports.

Example:

```text
53
67
68
8443
```

instead of

```text
domain
bootps
bootpc
https
```

---

## Example 11 — Show All Sockets

Command

```bash
ss -a
```

Meaning

Displays every socket on the system, including:

- TCP
- UDP
- UNIX
- Netlink
- ICMP
- Listening sockets
- Established connections

---

## Example 12 — Numeric Output

Command

```bash
ss -n
```

Meaning

Shows numeric port numbers instead of service names.

---

## Example 13 — Show Processes

Command

```bash
ss -p
```

Meaning

Displays the process using each socket.

Useful for identifying which application owns a connection.

---

## Example 14 — IPv4 Only

Command

```bash
ss -4
```

Meaning

Shows only IPv4 sockets.

---

## Example 15 — IPv6 Only

Command

```bash
ss -6
```

Meaning

Shows only IPv6 sockets.

---

## Example 16 — Filter by Source Port

Command

```bash
ss '( sport = :22 )'
```

Meaning

Displays sockets whose source port is **22 (SSH)**.

No output simply means no matching socket exists.

---

## Example 17 — Filter by Destination Port

Command

```bash
ss '( dport = :80 )'
```

Meaning

Displays sockets communicating with destination port **80 (HTTP)**.

---

## Example 18 — Established TCP Connections

Command

```bash
ss -tn state established
```

Meaning

Shows active established TCP connections.

---

## Example 19 — Established TCP with Process

Command

```bash
ss -tp state established
```

Meaning

Shows established TCP connections along with the owning process.

# 8. Interview Questions

**Q. What does SS stand for?**

A. Socket Statistics.

---

**Q. Why is SS preferred over netstat?**

A. It reads socket information directly from the Linux kernel, making it much faster.

---

**Q. What does `ESTAB` mean?**

A. Established connection.

---

**Q. What does `LISTEN` mean?**

A. The application is waiting for incoming connections.

---

**Q. What does `UNCONN` mean?**

A. Unconnected socket, commonly seen with UDP.

---

**Q. What does `-n` do?**

A. Displays numeric port numbers instead of service names.

---

**Q. What does `-p` do?**

A. Shows the process using the socket.

---

**Q. Why did `ss -t` show no output?**

A. Because no active TCP connections existed.

---

**Q. Why did `ss -u` show output?**

A. Background services like DNS, DHCP, or VPN were using UDP.

# 10. Summary

- `ss` displays network socket information.
- Faster than `netstat`.
- Supports TCP, UDP, IPv4, IPv6, UNIX and Netlink sockets.
- `-t` → TCP sockets.
- `-u` → UDP sockets.
- `-l` → Listening sockets.
- `-a` → All sockets.
- `-n` → Numeric ports.
- `-p` → Show owning process.
- `-4` → IPv4 only.
- `-6` → IPv6 only.
- Common socket states include `LISTEN`, `ESTAB`, and `UNCONN`.
- Widely used for network troubleshooting and service verification.
